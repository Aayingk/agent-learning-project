# Agent Learning Project - 需求与约束说明书

## 1. 项目概述

### 1.1 项目目标
构建一个**教学级Agent开发项目**，帮助开发者掌握Agent开发的核心技能，为求职agent开发岗位做准备。

### 1.2 目标用户
- **主要用户**：转行做Agent开发的学习者
- **次要用户**：面试官（通过项目评估候选人能力）

### 1.3 成功标准
- [ ] 代码可运行，demo可直接演示
- [ ] 覆盖Agent开发岗位的核心技术栈
- [ ] 代码结构清晰，易于讲解
- [ ] 每个核心功能都有可独立演示的测试用例

---

## 2. 功能需求

### 2.1 核心功能（MVP）

| 功能 | 优先级 | 描述 |
|------|--------|------|
| **多LLM后端支持** | P0 | 支持OpenAI/Claude/Ollama切换 |
| **工具调用** | P0 | Agent能调用外部工具（搜索、计算器、文件操作） |
| **短期记忆** | P0 | 对话历史管理，支持上下文窗口管理 |
| **RAG检索** | P1 | 基于文档的问答能力（向量化+检索） |
| **长期记忆** | P1 | 向量存储的长期记忆，支持语义检索 |
| **多Agent协作** | P1 | 多个Agent分工协作完成复杂任务 |

### 2.2 界面需求
- **前端**：Streamlit聊天界面（简单、快速）
- **后端**：FastAPI RESTful API（生产级接口设计）
- **CLI**：命令行界面（无依赖测试）

---

## 3. 功能边界

### 3.1 范围内
- Agent核心能力实现（工具、记忆、RAG、多Agent）
- 基础的工具集成（搜索、计算器、文件操作）
- 简单的文档向量化和检索
- 基础的多Agent协作模式（至少2个Agent）

### 3.2 范围外
- 复杂的工具生态（不集成几十种工具）
- 生产级部署（Docker、K8s不作为MVP）
- 复杂的多Agent编排（如自治团队、分层Agent）
- 实时流式输出（MVP支持即可，不做优化）

---

## 4. 接口规范

### 4.1 输入
```json
{
  "message": "用户输入的消息",
  "session_id": "会话ID",
  "llm_provider": "openai|anthropic|ollama",
  "agent_mode": "single|multi"
}
```

### 4.2 输出
```json
{
  "response": "Agent的回复",
  "tools_used": ["tool1", "tool2"],
  "reasoning": "推理过程（可选）",
  "tokens_used": { "prompt": 100, "completion": 200 }
}
```

### 4.3 错误处理
| 错误类型 | 处理方式 |
|----------|----------|
| LLM API错误 | 降级到备用LLM或返回友好提示 |
| 工具执行失败 | 记录错误，返回部分结果 |
| 超时 | 设置合理超时，返回进度 |
| 无效输入 | 参数校验，返回具体错误信息 |

---

## 5. 约束条件

### 5.1 技术约束
- **Python版本**：3.10+
- **主要框架**：LangChain + LangGraph
- **向量数据库**：ChromaDB（轻量，易部署）
- **内存存储**：内存（MVP）→ Redis（可选升级）

### 5.2 资源约束
- 本地可运行，无需复杂依赖
- API调用需要用户自备key
- 向量存储默认使用本地文件

### 5.3 时间约束
- MVP 2周内完成
- 每个功能模块1-2天
